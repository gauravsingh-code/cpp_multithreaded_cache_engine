RESP (Redis Serialization Protocol ) : Redis client (your application) and the Redis server use to talk to each other. 

DAEMON : A Daemon is simply a background process that runs silently to manage
             these two tasks without freezing the main database engine.
        - The background daemon quietly writes all that data to an .rdb file on the solid-state drive (SSD).
        - use the fork() mechanism of os to run in background

Snapshots (RDB) - Single file binary representation of entire dataset 
        - OS concept called Copy-on-Write (CoW), this child process gets a point-in-time duplicate of 
            the memory without actually copying the data in RAM.
        - Pros: The file size is very small and optimized. It is incredibly fast to load back into memory if the server restarts.
        - Cons: You can lose data. If you configure snapshots to happen every 5 minutes,
            and the server crashes at minute 4, you lose 4 minutes of data.

WAL(Write-Ahead Log (AOF)) - To solve the data-loss problem of snapshots, 
    Redis uses a Write-Ahead Log (WAL), which Redis calls the Append Only File (AOF).


RAII -  stands for Resource Acquisition Is Initialization. It is a core programming technique in C++ where the
         lifecycle of a resource is strictly tied to the lifetime of a local object.

2. WSAStartup(MAKEWORD(2, 2), &wsaData)
WSAStartup(...): The function that initiates the WS2_32.dll library for the process.
MAKEWORD(2, 2): A macro that constructs a WORD specifying the requested version of Winsock.
2, 2 requests Winsock Version 2.2 (the standard modern version for Windows network programming).
&wsaData: Passes the address of the WSADATA structure to receive configuration details populated by Windows.

3. Error Handling (if (...) != 0)
WSAStartup returns 0 on success.
If it returns any non-zero value, it indicates an initialization failure (e.g., system network subsystem not ready, requested version unsupported).

This restriction cascades up through any class that owns a non-copyable member by value:

When a class contains a non-copyable member, the compiler implicitly deletes its own copy constructor too.
